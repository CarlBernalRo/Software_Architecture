💻 Capa de Presentación

Es la "frontera" del sistema. Su única preocupación es la interacción con el mundo exterior.

  • Frontend: Vistas en React, manejo de rutas con react-router-dom.
  • Backend API: Controladores Express que capturan el req.body.
  • Validación: Usa Zod para asegurar que los datos tengan el formato correcto antes de entrar a la lógica.

⚙️ Capa de Aplicación

Aquí se define qué hace el sistema. Orquesta los procesos sin conocer los detalles técnicos.

  • Casos de Uso: Clases como AgendarCitaUseCase.
  • Orquestación: Coordina que el médico exista, que el paciente sea válido y que el horario se guarde.
  • Independencia: No sabe si los datos vienen de una web o de una app móvil.

🧠 Capa de Dominio

El corazón del negocio. Contiene las reglas que existirían incluso si no hubiera computadoras.

  • Entidades: Modelos con lógica propia (ej. Cita.cancelar()).
  • Invariantes: Un médico no puede ocupar dos espacios al mismo tiempo.
  • Contratos: Interfaces que obligan a la infraestructura a cumplir ciertos métodos.

💾 Capa de Infraestructura

Los detalles técnicos y herramientas externas que el sistema necesita para funcionar.

  • Persistencia: Implementaciones de SQL para MySQL usando mysql2.
  • Seguridad: Configuración de CORS y variables de entorno dotenv.
  • Consumo API: Clientes HTTP para conectar el Frontend con el Backend.
